# Changelog

All notable changes to the Conventional Commits plugin will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2025-01-18

### Added

- Complete Conventional Commits v1.0.0 specification skill
- Comprehensive commit type reference (feat, fix, Angular convention types)
- Semantic versioning correlation guide
- Validation patterns with regex and Python examples
- Breaking change guidance with multiple methods
- Tool integration documentation (commitlint, pre-commit, semantic-release, git-cliff)
- 15+ concrete usage examples
- Best practices for description formatting and body structure
- README with installation instructions and quick start guide
- Extensive examples documentation covering common scenarios
- References to official specifications and related tools

### Documentation

- Created comprehensive README.md with features, installation, and usage
- Added docs/examples.md with 15 real-world usage scenarios
- Included references to Conventional Commits v1.0.0 specification
- Added tool integration guides for commitlint, semantic-release, and git-cliff
